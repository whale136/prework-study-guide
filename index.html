HTML

  • HTML gives content structure.
  • The head element contains information about the webpage.
  • The body element represents the visible content shown to the user.
  • Elements in header are not visible to user.
  • ul: unordered lists. ol: ordered lists. li: list items. With opening and ending tags.

CSS

  • CSS gives the content style.
  • A margin indicates how much space we want around the outside of the element.
  • A padding indicates how much space we want around the content inside an element.
  • There are two versions of the Box Model known as the Standard and Alternative.
  • The three versions of CSS are Inline, Internal Style Sheet, and External Style Sheet.
  • Inline refers to specific HTML tags, making for quick and permanent change.
  • Internal contains CSS rules in the head section of HTML, which only apply to that page/
  • External allows the user to keep all CSS rules in a separate file, being the most useful.

Git

  • git status: check what branch we are currently in.
  • git checkout branch-name: switch to said branch.
  • git checkout -b branch-name: creates a new branch and switches to it.
  • git pull origin main: pull changes from origin(remote) to main(base) branch.
  • git push origin branch-name: push changes from our local repo.
  • git add -A: stages our modifications/commits our changes.

JavaScript

  • JavaScript handles the logic and allows users to interact with webpage.
  • Control flow is the order in which a computer executes code in a script.
  • Functions allows us to call code that would not be run regularly.
  • A variable is a named container that allows us to store data in our code.
  • JS is a weak typed language which allows implicit type conversion when an operation involves mismatched types.
  • Array's are used to store groups of data to a single variable.
  • For loops repeat until a specified condition returns a false.
